Business Use Case Background/Description

This sample-project (stand-alone static web application) represents a side-project developed while working with Capital One. A workflow consists of multiple "Actions" that can be tailored by the user. This feature saved the company money and time by automating the process via UI. Previously, each Workflow request sent by an employee required manual insertion.

Technical Details Composition/Implementation

This project uses Angular (with Angular Material) in conjunction with Stackblitz (for demo purposes) as the front-end framework. In production, a real Capital One API was used to fetch/post/update Business and/or Adhoc workflows. For legal reasons, this demo uses mock data in place of live data that was used in production. The Angular Material Drag and Drop feature was implemented with Reactive Forms and Angular Material Stepper to simulate a more facile approach to submitting a Workflow. Sorting/Validation logic is implemented behind the scenes to demonstrate proper Form implementation. This project was not designed for mobile-use, hence, the lack of reponsiveness.
